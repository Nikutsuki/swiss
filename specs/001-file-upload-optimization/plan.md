# Implementation Plan: Optimize file upload UX and performance

**Branch**: `001-file-upload-optimization` | **Date**: 2026-03-28 | **Spec**: [/specs/001-file-upload-optimization/spec.md](/specs/001-file-upload-optimization/spec.md)
**Input**: Feature specification from `/specs/001-file-upload-optimization/spec.md`

## Summary

This plan addresses performance bottlenecks and UX limitations in the file transfer workspace. Key improvements include dynamic chunking for large files, a multi-file transfer UI with dismissal capabilities, and advanced telemetry with smoothed speed and ETA calculations.

## Technical Context

**Language/Version**: TypeScript (Next.js 15+ / React 19)
**Primary Dependencies**: React, `@swiss/webrtc-signaling`
**Storage**: N/A (P2P Memory/DataChannels)
**Testing**: Vitest for utility logic, Playwright for E2E transfer scenarios
**Target Platform**: Web (Modern Browsers supporting WebRTC)
**Project Type**: web-service (frontend of the P2P drop service)
**Performance Goals**: Increase large file (>100MB) transfer speed by 20%+ via optimized chunk sizes.
**Constraints**: <500ms UI latency for state updates; prevent memory exhaustion during large transfers via `bufferedAmount` backpressure.
**Scale/Scope**: Focus on `apps/monolith-drop` and `packages/webrtc-signaling`.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **I. Security and Privacy by Default**: P2P transfers are already DTLS encrypted. No changes to the signaling path. Ensure that speed/ETA calculations don't leak metadata. [PASS]
- **II. Contract-First Interfaces**: No changes to the signaling API. P2P `file-frame` protocol remains backwards compatible (chunk size is a sender-side optimization). [PASS]
- **III. Testable Delivery Gates**: Each improvement (chunking, multi-file UI, telemetry) is independently testable. [PASS]
- **IV. Observability and Operability**: Telemetry improvements *are* observability. Ensure smoothing doesn't mask underlying transport failures. [PASS]
- **V. Keep It Small and Reversible**: UI changes are isolated to the session workspace. [PASS]
